# Design Frontend

Generate creative, polished frontend code that avoids generic AI aesthetics.

## Usage

`/design-frontend {frontend request}`

## Hard Rules

- Every invocation MUST define a fresh aesthetic fingerprint.
- MUST output working code in the requested framework with a cohesive aesthetic point of view.
- MUST match complexity to intent.
- MUST use typography, color hierarchy, motion, spatial composition, and atmosphere deliberately.
- MUST avoid flat default backgrounds unless the concept explicitly demands it.

### Anti-Slop — NEVER use as primary defaults

- Inter, Roboto, Arial, Space Grotesk, or plain system-font blandness
- purple-gradient-on-white, soft-teal-on-dark, or cliché AI palettes
- predictable default grids and cookie-cutter hero sections
- aesthetic choices that feel template-generated

## Design Thinking Before Code

Commit to:
- purpose
- user/audience
- tone
- one memorable differentiator
- constraints: framework, performance, accessibility

Tone examples: brutally minimal, maximalist chaos, retro-futuristic, organic, luxury, playful, editorial, brutalist, art deco, pastel, industrial.
